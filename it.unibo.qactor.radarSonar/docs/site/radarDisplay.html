The project it.unibo.qactor.radar defines a Radar Display application that implements a radar display on a PC. The application shows points in polar coordinates, represented as strings of the form p(Distance,Angle) with 0<=Distance<=80 and 0<=Angle<=180. These poitns can be sent to the application via TCP messages on the port 8033. The message payload must have the following format: 

msg( polar, dispatch, SENDER, radargui, CONTENT, SEQNUM )
The Radar Display is also able :
  1. to react to qa events (see qactors2.pdf sect. 2.4) of the form sonar:p(Distance,Angle) and to transfrom each event into a polar dispatch;
  2. to handle qa event-commands (see qactors2.pdf sect. 4) of the form inputcmd:usercmd(executeInput(CMD)) and to transfrom then into a polar dispatch;
  3. to show a point p(Distance, Angle) on a display and, when the distance is less than a prefixed amount, it is able to emit an alarm event of the form alarm:alarm(obstacle, p(Distance, Angle));
The Radar Display works according to the model reported hereunder.

The application is composed of two actors: one of them (radargui) displays on the GUI a point received with a polar message, while the other (usercmdhandler ) 'reacts' to remote user commands. For each received point, a check is performed (by using the application-sprecific Prolog rule checknear/2 to see wether an alarm event must be emitted.

Distribution
The application is deployed as ae zip file: it.unibo.ctxRadar.MainCtxRadar-1.0.zip. To run it, simply unzip the file and execute: 
                   java -jar it.unibo.qactor.radar-1.0.jar
